# Neeeead 文案库 (Copywriting)

这里集中维护项目中所有的随机（N选1）动态文案，方便文案策划或开发者统一查看和管理。

---

## 文案治理规则（新增）

### 1. 文件职责

- 本文件只维护文案资产，不维护业务逻辑。
- 业务逻辑与触发条件在 `product.md` 定义。
- 技术接入与 key 映射在代码中实现。

### 2. 文案分层

- `随机文案`：同一场景 N 选 1，增强活性。
- `固定文案`：按钮、标题、提示语等必须稳定一致。

### 3. Key 命名建议（代码侧）

- 推荐格式：`module.scene.state`，例如：
  - `canvas.validation.invalidShape`
  - `canvas.eyePrompt.count0`
  - `auth.modal.subtitle`

### 4. 变更规范

- 每次改文案必须标注触发条件。
- 涉及用户路径文案时，同步检查 `product.md` 是否需更新。
- 避免在多个文件重复维护同一句文案。

## 1. 图形合规检测失败提示 (5选1)
> 触发条件：生成的图形出现自交、破裂或空洞时，禁用“生成卡片”按钮并在 UI 上红字提示。

- 这只捏物似乎有些破碎，需要一点点修补
- 警告，几何结构发生了坍缩
- 你这只捏物好像还没发育完全
- 你这捏的什么玩意儿，再调一个版本
- 唔，这只长得不大对劲，要想想办法

## 2. 眼睛点缀动态交互提示 (各状态5选1)
> 触发条件：图形结构合规后，根据用户在画布上点击添加的眼睛数量，动态在 UI 上切换对应的随机提示语。

### 0只眼睛 (初始空洞状态)
- 嗯，似乎还少了点什么
- 我看不见，帮帮我
- 至少点上一只眼睛啊喂
- 捏物不捏眼，等于没捏眼
- 眼睛呢，请问

### 1只眼睛 (刚刚点上)
- 嗯嗯，能看见啦
- 你好像懂了
- 谢谢
- 所以呢
- 哦

### 2只眼睛 (双目对视)
- 你太懂了
- 哦哦哦
- 看，立体视觉了
- 你好聪明啊
- 对视中

### 3只眼睛 (最高境界)
- 卧槽
- 你个神人
- 给你懂完了
- 嗷嗷嗷
- 多多益善
