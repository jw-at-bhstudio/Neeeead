# 盒中捏物 (Neeeead) - 产品文档

## 产品愿景
一个结合了心理测试与生成艺术（Generative Art）的社交化 Web 应用。用户通过互动或调参生成代表自己性格与状态的专属“有机几何生物”（捏物），并能将其放入一个公共的虚拟生态“盒子”中，与其他捏物互动，或领养至私人盒子中进行养成。

---

## 信息架构（当前统一方案）

### A. 未登录用户

- `捏只新的`（问卷 -> 顺手捏/大师捏）
  - 可完整体验生成流程
  - 不可保存到「我的盒子」
  - 不可发布到「盒子广场」
- `盒子广场`
  - 可浏览公开捏物（只读）

### B. 已登录用户

- `我的账号`
  - 账号基础信息与退出登录
- `我的盒子`
  - 管理自己的捏物（草稿/公开等状态切换）
- `盒子广场`
  - 浏览公开捏物
- `捏只新的`
  - 可保存到我的盒子
  - 可发布到盒子广场

## 核心模块状态与功能定义

### 1. 问卷输入模块 【已上线】
通过感性问题映射理性生成参数，让用户在心理投射中创造形状。
- **题目映射**：
  - 社交圈广度 -> **顶点数量 (Vertices)**
  - 生活计划性 -> **不规则度 (Irregularity)**
  - 内心纠结度 -> **复杂度 (Complexity)**
  - 社交直接性 -> **描边厚度 (Stroke Offset)**
- **交互形式**：使用隐藏刻度的无极滑动条，用户仅凭直觉（如“少而精” vs “广而多”）进行选择。
- **品牌预设模式 (Brand Preset)**：将最终生成的参数严格限制在美学安全范围内，确保产出的图形符合品牌调性。

### 2. 生成图形模块 【已上线】
应用的核心生成引擎，提供实时渲染与高级编辑能力。
- **实时 SVG 渲染**：利用 RDP 路径简化与贝塞尔曲线平滑算法，实时将参数转化为有机的生物形态。
- **高级开发者模式 (Developer Mode)**：允许用户直接拨动底层滑块（0-400顶点、0-1000描边等）进行极限测试与艺术创作。
- **智能缩放与防裁切 (Fit Mode B)**：无论图形如何扭曲、描边如何膨胀，渲染引擎会自动计算精确的外包围盒（BBox）并将图形等比缩放、完美居中于画布内。
- **图形拓展 (Expand Shape)**：利用 Web Worker 进行后台像素级轮廓追踪，将带粗描边的线框转化为可导出的纯粹“矢量面”（Solid Vector Fill）。
- **点睛交互**：支持在画板上点击，为捏物点上最多三只“眼睛”。

### 3. 卡片导出模块 【已上线】
生成高分辨率的品牌分享海报，便于社交裂变。
- **尺寸规范**：1080x1440 (3:4 竖版)，全画幅 40px Margin。
- **高清离屏渲染**：使用 Canvas 2D 重新绘制 SVG 结构。
- **信息展示**：
  - 标题：“我的捏物”（未来支持自定义及 AI 命名）。
  - 创作者署名与编号（预留给注册盒子居民）。
  - 声音拟态设定（预留扩展）。
  - 核心几何参数的数值透出（Vertices, Irregularity, Complexity, Stroke）。
- **品牌元素**：底部预留二维码位，底部居中带有版权声明（© 2025~2026 四百盒子社区 & 不含观点）。

### 4. 盒子空间模块 【计划中】
一个所有公开发布的捏物共同生活的公共 3D/物理生态环境。
- **广场展示**：获取 Supabase 中公开状态的捏物参数，实时生成并在公共“缸体”内渲染。
- **物理碰撞**：捏物们作为软体对象（Soft Body）或 2D 刚体，在空间内进行自然移动和碰撞。
- **实时互动**：用户点击投放食物，所有在线用户通过 WebSocket 实时看到食物掉落，捏物们产生争抢聚集行为。

### 5. 我的盒子模块 【计划中】
用户的个人管理中心与展示柜。
- **账户体系**：支持第三方登录与用户 Profile 建立。
- **领养机制**：在公共盒子中看到喜欢的捏物，可点击“领养”并转移归属权至私人盒子。
- **展示画廊**：以瀑布流或网格形式展示用户历史创作或领养的全部捏物。

### 6. 捏物养成模块 【计划中】
为几何图形赋予生命周期和状态变化。
- **数值系统**：引入饱食度、快乐值、年龄等动态状态（Stats）。
- **形态演化**：随着养成状态的变化，捏物的核心参数（如圆滑度、不规则度）会发生缓慢渐变（如：饿了会变得尖锐，吃饱了会变得圆润）。
- **行为互动**：允许用户对特定捏物执行“喂食”、“抚摸”等动作，并在公共/私人盒子中触发对应的动画反馈。

## 版本迭代优先级（执行顺序）

### Phase 1（当前）

- 完成用户登录与保存捏物
- 完成 RLS 与基础数据表稳定
- 完成未登录/已登录权限分流

### Phase 2（紧接）

- `我的盒子` 最小可用：列表、删除、状态切换（private/public_pool）
- `盒子广场` 最小可用：公开列表瀑布/网格展示

### Phase 3（上线）

- GitHub 主分支稳定
- Vercel 首发上线
- 线上回归测试（登录、保存、发布、可见性）

---

## 模块边界说明

- `捏只新的`：负责生成与编辑，不承担账号管理。
- `我的账号`：只做身份相关功能（登录状态、退出、资料）。
- `我的盒子`：只做“我的数据”管理，不显示他人私有内容。
- `盒子广场`：只读公开内容，不展示私有草稿。
